Music Discovery AI

แอปพลิเคชันค้นหาเพลงอัจฉริยะที่มาพร้อม UI สองรูปแบบ, การผนวก AI, และกระบวนการ CI/CD ที่สมบูรณ์

ภาพรวมโครงการ (Project Pitch)

เอกสารนี้นำเสนอภาพรวมของโปรเจกต์ Music Discovery App โดยปรับปรุงเนื้อหาให้สอดคล้องกับผลลัพธ์สุดท้ายของโปรเจกต์ ซึ่งเน้นการพัฒนาแอปพลิเคชันที่มีส่วนติดต่อผู้ใช้แบบกราฟิก (GUI), การผนวก AI, และการทำ CI/CD และ Cloud Deployment ที่เป็นมาตรฐานอุตสาหกรรม

ข้อมูลโครงการ รายละเอียดตามผลลัพธ์ของโปรเจกต์
ประเภทโครงการ Desktop GUI และ **Cloud-Deployed** Web Application
ฟังก์ชันหลัก ค้นหาเพลง/ศิลปิน, สร้าง Playlist, **รับบทสรุป Playlist ด้วย AI**, และส่งออกเป็นไฟล์ CSV
แนวคิดหลัก #1 **Multi-Platform Development & API Integration:** พัฒนาแอปพลิเคชันที่ใช้งานได้ทั้งบน Desktop (Python) และ Web (JavaScript) โดยทั้งสองมีการเรียกใช้ API ภายนอก (Last.fm, Google Gemini) และประมวลผลข้อมูล JSON
แนวคิดหลัก #2 **Object-Oriented & Modular Design:** ออกแบบโดยแยกส่วน Logic หลัก (`music_app.py`) ออกจากส่วนการแสดงผล (`desktop_app.py`, `index.html`) อย่างชัดเจน
แนวคิดหลัก #3 **Modern DevOps & Cloud Deployment:** สร้างกระบวนการ CI/CD Pipeline อัตโนมัติด้วย GitHub Actions เพื่อ build Docker image และทำการ Cloud Deployment ไปยัง Render
แนวคิดส่วนขยายที่เป็นไปได้
  • การบันทึก Playlist บน Cloud (เช่น Firebase)
  • การเชื่อมต่อกับบริการ Streaming (เช่น Spotify)
  • ปรับปรุงระบบแนะนำเพลงให้แม่นยำยิ่งขึ้น

สถาปัตยกรรมและเทคโนโลยี

แผนการพัฒนา (Development Plan)

การพัฒนาถูกแบ่งออกเป็น 4 เฟสหลัก ตั้งแต่การเตรียมความพร้อม, การสร้างแกนหลักของโปรแกรม, การพัฒนาส่วนติดต่อผู้ใช้, ไปจนถึงการเพิ่มฟีเจอร์ขั้นสูงและการนำขึ้นสู่ระบบคลาวด์ คลิกที่แต่ละเฟสเพื่อดูรายละเอียด

P-0: Artifact Readiness (1 Hour)

เตรียมความพร้อมของโปรเจกต์และ Dependencies

  • ตรวจสอบ Python Environment
  • สร้าง `requirements.txt` และติดตั้ง Dependencies
  • สร้างโครงสร้าง Directory แบบ Flat Structure
  • เริ่มต้น Version Control ด้วย Git และสร้าง `.gitignore`

P-1: Foundation & Core Logic (8 Hours)

สร้างแกนหลักของแอปพลิเคชัน (OOP Models) และเชื่อมต่อ API

  • กำหนด Data Models (คลาส Track, Artist)
  • สร้าง API Client เพื่อเชื่อมต่อ Last.fm API
  • พัฒนา Playlist Manager Logic
  • แก้ไขปัญหาการ Export ไฟล์ CSV ให้รองรับภาษาไทย

P-2: UI/UX Development (10 Hours)

พัฒนาส่วนติดต่อผู้ใช้ (UI) สองเวอร์ชัน

  • สร้าง Desktop App UI ด้วย ttkbootstrap ให้สวยงาม
  • เชื่อม Desktop App เข้ากับ Core Logic
  • สร้าง Web App UI แบบ Single-Page ด้วย HTML/JS/TailwindCSS
  • พัฒนา Logic ในฝั่ง JavaScript เพื่อให้ Web App ทำงานได้สมบูรณ์

P-3: Advanced Features & Deployment (8 Hours)

เพิ่มฟีเจอร์ AI, สร้าง CI/CD Pipeline, และ Deploy ขึ้นสู่ Cloud

  • **AI Integration:** เพิ่มฟีเจอร์ "Generate AI Summary" โดยเชื่อมต่อกับ Google Gemini API
  • **CI/CD Pipeline:** สร้าง Workflow ใน GitHub Actions เพื่อ build Docker image และ push ไปยัง GHCR โดยอัตโนมัติ
  • **Cloud Deployment:** นำ Docker image ไป Deploy บนแพลตฟอร์ม Render ให้สามารถเข้าถึงได้ผ่าน Public URL
  • **Final Documentation:** อัปเดต `README.md` ให้สมบูรณ์พร้อม Live URL
